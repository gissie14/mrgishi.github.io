Hugo × GitHub PagesでBlogを作成してみた

Published On: 2020-07-26, Reading Time: 1 minute

実現したいこと

  • GitHub で管理するブログ構築

導入理由

現在、WordPress ではない、手軽なブログ管理システム、運用方法を模索しています。 そんな中、GitHub のレポジトリで管理できる方法を発見したので、試しに使ってみようと考えました。

利点

補足

Hugo の Sugoi テーマを利用

cf. Sugoi ｜ Hugo Themes： https://themes.gohugo.io/hugo-sugoi/

選択理由

  • UI が使いやすそうだった

欠点

  • カテゴリページも存在するが分かりづらい
  • デフォルトでメールフォームが存在するが初期設定では使えない
  • 更新や管理は楽だが、コード全体の管理が美しくない（静的サイトの苦手な点）

既存のテーマでは、これだ！というものはありませんでした。 テーマの自作もできるので、時間のあればオリジナルのテーマを作成することも考えます。

導入方法

下記ブログを参照。

cf. GitHub Pages と Hugo でブログをつくった｜ meow.md： https://uzimihsr.github.io/post/2019-08-07-create-blog-1/

結論

静的サイトは更新や記事の管理がしやすい、表示の高速化や強固なセキュリティといった利点があるものの、各ページの HTML ファイルが書き出されてしまうのは美しくないと感じました。 吐き出させるコードが気になってしまうので、正直ずっと使っていきたいとは思えなかったです。
